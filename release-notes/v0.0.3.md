# Fabrik v0.0.3

## Fixes

### Deep-fetch rate limit fix (#190)
- Done items with no worktree are no longer deep-fetched on every poll
- Cleanup stages skip deep-fetch entirely (don't need comments)
- Adds completion label when no worktree exists, permanently skipping the item
- Reduces GraphQL consumption from ~32,000 to ~500 points/hour on large boards

### Release auto-upgrade in multi-repo mode (#188)
- Renamed `primaryWorktrees` to `devCheckout` — clarifies it's about Fabrik's own source, not the user's project
- Removed spurious worktree guard from `checkReleaseUpgrade` — release upgrades don't need a worktree
- `--auto-upgrade` now works correctly for release binaries in multi-repo job-control directories

## Upgrading

```bash
# From a previous release binary
the download command below

# Or download directly
gh release download --repo tenaciousvc/fabrik --pattern '*.tar.gz' -O - | tar xz
```


> **Note:** Binary self-update via `fabrik upgrade` works from v0.0.8 onwards. For this version, download manually:
> ```bash
> gh release download --repo tenaciousvc/fabrik --pattern 'fabrik_*_darwin_arm64.tar.gz' -O - | tar xz
> ```
