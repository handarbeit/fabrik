# Fabrik v0.0.71

## Summary

This release lands the **internal merge train** — Fabrik can now batch multiple ready PRs into a single trial branch, validate them together, and land them as one integration merge, collapsing the O(N²) strict-mode retest cascade that made large queues expensive. Red batches are isolated by halving bisection: the poisoner is ejected and the survivors still land, and a runaway guard pauses a repo that churns without landing so a bad batch can't burn your CI budget. On repositories where GitHub's native merge queue is available, Fabrik composes with it (ADR-058); everywhere else it runs its own train (ADR-059), selected automatically per repo.

Alongside the train, this release makes CI gating trustworthy under pressure: a freshly-pushed commit whose CI is still running is never misread as failed, so yolo issues no longer get spuriously paused mid-validation. Stage-drift warnings became value-aware (no more nagging about default knobs that change nothing), and several engine paths were hardened to survive GitHub rate-limit exhaustion without losing work.

## Features

- **Internal merge train (ADR-059)** — batch ready PRs into a trial branch, validate combined, land as one integration PR; halving bisection ejects a poisoner while survivors land; worker-driven inline conflict resolution; per-repo opt-in via `merge_train` (#924)
- **Merge-train runaway guard (ADR-059 D8)** — rolling per-repo trial counter pauses all Queued members after too many trials without a land; tunable via `--max-train-trials-per-window` / `--train-trial-window` (and matching env vars)
- **Native merge-queue integration (ADR-058)** — on repos with GitHub's merge queue enabled, Fabrik enqueues rather than direct-merging; the `Queued` stage auto-selects native queue vs. internal train per repo (#924)
- **Value-aware stage-drift** — drift warnings and `refresh-stages` no longer flag a missing default key (e.g. `kill_grace`) when adding it would change nothing; a genuinely meaningful default still warns (#977)

## Fixes

- **CI-fix reinvoke no longer spuriously pauses on pending CI** — pending checks on the current head always beat a stale failure; no-op reinvokes don't consume cycle budget; a stale cached failure is re-validated live before it can block (#958)
- **Merge-train poison-well** — ejected/paused members are excluded from batch snapshots and their issues are closed on landing, ending the endless red→bisect→eject reform loop that burned thousands of Actions runs (#963, #965)
- **No-work-needed decision is now durable** — a `FABRIK_NO_WORK_NEEDED` outcome is recorded before the board move, so a rate-limited move failure retries instead of silently re-running the whole pipeline (#981)
- **Cross-repo child spawn survives a stale comment snapshot** — `preImplement` re-reads live when the Plan comment is missing, rather than silently skipping the spawn (#982)
- **Startup no longer warns** about the `Backlog`/`Done` columns as unmatched board columns

## Improvements

- Stage-drift warnings now name the missing key(s) in the summary title (`<file> missing N field(s): <keys>`) instead of only counting them
- Merge-train cost caps (`max_batch_size`, bisection-validation budget) and base-SHA-pinned train worktrees
- Poll-driven cache reconcile runs unconditionally (not gated on webhooks), so a webhook-less deployment still self-heals label drift

## Internal

- Overhauled the e2e harness: full clean-slate `reset.sh` (board items + PRs + branches), `run.sh --clean`, and a parallelism cap to stop the shared bed from oversubscribing; extensive merge-train e2e coverage (bisection, runaway guard, restart safety, happy-path landing).

## Upgrading

```bash
# Auto-upgrade from a running Fabrik instance
# Fabrik checks for new releases each poll cycle and upgrades automatically with --auto-upgrade

# Or download directly (auto-detects OS and architecture)
gh release download --repo handarbeit/fabrik \
  --pattern "fabrik_*_$(uname -s | tr A-Z a-z)_$(uname -m | sed 's/x86_64/amd64/' | sed 's/aarch64/arm64/').tar.gz" \
  -O - | tar xz
```
