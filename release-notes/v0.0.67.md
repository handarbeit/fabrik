# Fabrik v0.0.67

## Summary

Three themes:

1. **Cross-repo decomposition is now fully autonomous.** The v0.0.66 spawn machinery had two follow-on issues that blocked end-to-end autonomy: the first spawn into a never-touched repo failed silently because the engine hadn't lazy-cloned it yet (#803), and spawned sub-issues landed with neither `fabrik:yolo` nor a stage status so they needed manual triage before processing could continue (#806). Both fixed. A yolo'd cross-repo issue now flows through both repos in coordination with zero operator intervention.

2. **Worktree boundary audit is opt-in.** The cross-repo ref audit that shipped in v0.0.66 produced false positives in real-world setups — any routine `git fetch` against a sibling bare clone would update its `refs/remotes/origin/*` and the audit would flag it as "Claude mutated state outside the worktree." The audit is now gated behind a new `worktree_boundary_audit` config flag, default OFF (#810). When enabled, the snapshot correctly filters to `refs/heads/*` + `refs/tags/*` only and uses a whitelist guard for non-standard ref namespaces (#808). Auto-upgraders immediately stop hitting the false positive.

3. **End-to-end test harness.** New `tests/e2e/` package with five scripted scenarios (single-repo dispatch, full pipeline, `FABRIK_NO_WORK_NEEDED` short-circuit, `FABRIK_BLOCKED_ON_INPUT` pause+resume, cross-repo spawn). Runs in parallel against a real Fabrik instance and the canonical test repos — ~36 min wall-clock for the full suite (vs ~90 min sequential). Every regression that escapes a release should earn a new scenario here.

## Features

- **Spawn-target repos initialized on-demand** (#803). Pre-Implement spawn step now lazily clones and registers a `WorktreeManager` for any repo the parent's Plan emits in a `FABRIK_SPAWN_CHILD_BEGIN` block, regardless of whether Fabrik has processed an issue from that repo before. First-spawn-into-untouched-repo now works.
- **Sub-issues inherit parent's status and `fabrik:yolo`** (#806). The pre-Implement spawn step sets the child's project Status to `Specify` (or the first non-Backlog processing stage) and carries over `fabrik:yolo` / `fabrik:cruise` from the parent. Cross-repo work that the operator already authorised flows autonomously instead of pausing at the boundary.
- **`worktree_boundary_audit` config flag** (#810). Default OFF. Enable explicitly to turn on the Layer 2 cross-repo ref audit; off by default until the false-positive surface is fully characterised.
- **End-to-end integration test harness** (`tests/e2e/`, #804). Scenario-driven, parallel-capable, with a runner script (`scripts/e2e/run.sh`) and a state-reset script. Five scenarios covering dispatch, full pipeline, NO_WORK_NEEDED, BLOCKED_ON_INPUT, and cross-repo spawn. Gated behind the `e2e` build tag.
- **`addBlockedBy` GraphQL mutation fixed** (#800). v0.0.66 shipped with a non-existent mutation name (`addBlockedByIssue` / `blockedById`); the real GitHub API is `addBlockedBy` / `blockingIssueId`. Without this, cross-repo decomposition's parent→child linkage failed at runtime. Now correct.

## Fixes

- **Boundary audit excludes `refs/remotes/`** (#808). `snapshotRepoRefs` now filters to `refs/heads/*` and `refs/tags/*`, so routine fetches against sibling bare clones no longer trigger the false-positive "Stage Implement mutated refs in repositories outside its assigned worktree" failure. Companion whitelist guard for non-standard ref namespaces (notes, stash, etc.).
- **Spawn step logs warning when `statusField` is nil**. Defensive: prevents a silent skip when the project's Status field hasn't been resolved yet.
- **Documentation accuracy**: fabrik plugin skill's `blocked_by` wiring guidance corrected; README replaced `FABRIK_DECOMPOSED` reference with the current `FABRIK_SPAWN_CHILD_BEGIN/END` mechanism.

## Improvements

- `OrderedOptionNames` on `StatusField` — supports ordered fallback when resolving project Status options (foundation for #806's status-set in spawn).
- `resolveSpecifyOptionID` helper extracted for reuse by the spawn status-set path.

## Internal

- Release tooling: per-version release notes now archived in-tree under `release-notes/<tag>.md` (no more repo-root `release-notes.md`); `scripts/cut-release.sh` automates commit/tag/push as `@arbeithand` with explicit identity verification and a hard fail if the release author comes back as anyone else. Customised Specify skill no longer references fantasy-specific examples.
- Test improvements: `tests/e2e/` parallel-enabled with `t.Parallel()`; corrected misleading comment in `TestResolveSpecifyOptionID_SingleColumn`; Gemini and Copilot review feedback addressed throughout.
- Documentation: `docs/llms-full.txt` regenerated; spawn per-child mutation sequence documented; `worktree_boundary_audit` flag documented as opt-in.

## Upgrading

```bash
# Auto-upgrade from a running Fabrik instance
# Fabrik checks for new releases each poll cycle and upgrades automatically with --auto-upgrade

# Or download directly (auto-detects OS and architecture)
gh release download --repo handarbeit/fabrik \
  --pattern "fabrik_*_$(uname -s | tr A-Z a-z)_$(uname -m | sed 's/x86_64/amd64/' | sed 's/aarch64/arm64/').tar.gz" \
  -O - | tar xz
```
