# Fabrik v0.0.69

## Summary

Five themes:

1. **Critical hotfix for v0.0.68 supplant regression.** PR #854 introduced an unconditional cancel of the in-flight per-issue context whenever a poll cycle re-encountered an active worker. Every stage's own `stage:X:in_progress` label write fired a webhook, marked the cache stale, triggered a re-poll, and cancelled the worker â€” preventing any stage from completing a turn. Auto-upgrade now resolves to a binary that just skips in-flight items as it should.

2. **Stuck-Validate self-heals.** Two complementary mechanisms close the worst operator-pain category: a new `fabrik:revalidate` label (#856) collapses recovery to a single label apply, and SHA-change invalidation (#857) automatically re-enters Validate when a PR's HEAD SHA changes after stage:Validate:complete. Force-push to recover a stuck PR is now sufficient â€” no label juggling, no SIGHUP.

3. **Worktree lifecycle hardening.** `isProbeOnlyTerminal` no longer false-marks closed items terminal when their worktree still exists on disk (#858), and a periodic janitor scans + reaps orphaned worktrees on startup and every hour (#859). The "stranded worktree after restart between Validate complete and Done dispatch" failure mode is closed.

4. **CI gate handles Commit-Status-gated projects.** When `mergeable_state` actively blocks merge but no `check_runs` are visible (typical when branch protection requires a legacy Commit Status), the gate no longer falsely clears as "no CI configured" (#855). Inline CIWaitTimeout still escalates so the issue eventually pauses rather than blocking forever.

5. **Signal escalation contract for child processes (#853).** Stage process groups now receive SIGINT â†’ 10s grace â†’ SIGTERM â†’ 10s grace â†’ SIGKILL instead of immediate SIGTERM. Grace windows are per-stage configurable. Every signal logs structured `[#N kill] sending <signal> to PID <pid> (reason=...)` with reasons including `max_wall_time`, `inactivity_timeout`, `supplant_by_new_invocation`, `daemon_shutdown`. Test-wrapper contract documented in `docs/stage-lifecycle.md`.

## Features

- **`fabrik:revalidate` operator label** (#856). One-action recovery for stuck-Validate items. Apply the label and the engine clears `stage:Validate:complete`, `fabrik:paused`, `fabrik:awaiting-input`, `fabrik:awaiting-ci`, `fabrik:auto-merge-enabled`, then re-enters Validate on the current HEAD SHA. Idempotent and scoped to Validate stage; no-op-with-warning on other stages.
- **SHA-change invalidation for Validate completion** (#857). New `ValidateCompletedSHA` field in itemstate tracks the SHA at which Validate completed. On `PRHeadSHAUpdated`, if the new SHA differs from the recorded completion SHA, the engine clears `stage:Validate:complete`, `fabrik:auto-merge-enabled`, `fabrik:awaiting-ci`, `fabrik:awaiting-review`, and dispatches Validate on the next poll. Force-push triggers automatic re-Validate; legacy items with empty completion SHA fall back to `fabrik:revalidate`.
- **Periodic worktree janitor** (#859). New `runWorktreeJanitor` scans `.fabrik/worktrees/<owner>-<repo>/issue-N` on startup and every hour (configurable via `--janitor-interval`). Reaps worktrees whose issue is closed and either off-board or at a cleanup stage with `stage:Done:complete` and no in-flight worker. Respects dirty worktrees and registered workers; conservative when in doubt.
- **Signal escalation gradient with grace windows** (#853). New `--kill-grace-sigint` / `--kill-grace-sigterm` flags (also `FABRIK_KILL_GRACE_*` env vars, 10s default each) and per-stage `kill_grace: {sigint: ..., sigterm: ...}` YAML. Default `kill_grace` added to Implement/Review/Validate stage YAMLs. Reason propagation via `killReasonHolder` + `issueCtxs` map.
- **CI gate `mergeable_state` Commit-Status guard** (#855). When `len(check_runs) == 0` and `hadChecks == false` and `mergeable_state âˆ‰ {"", "unknown"}`, gate blocks rather than clearing. Inline `CIWaitTimeout` escalation removes `fabrik:awaiting-ci` and returns `timedOut=true` so the issue pauses cleanly when the block persists.
- **Paused-item CI recovery loop (R4)** (#845). Items paused with `fabrik:awaiting-ci` are now auto-recovered when their linked PR merges â€” separate from the main catch-up loop, uses `e.client.FetchLinkedPR` (not the cache) to avoid stale state. Clears the gate and pause labels, advances to Done without manual unpause.
- **CI gate handles merged/closed PRs and required-never-running checks** (R1/R2/R3/R5, #845). `R1`: merged PR clears gate immediately. `R2`: closed-not-merged pauses with explanatory comment. `R3`: required check that never runs (e.g. converted to `workflow_dispatch`) pauses via `pauseForRequiredNeverRunningCheck` after CIWaitTimeout dwell.
- **TUI pre-flight warnings panel** (#843). Persistent warnings file (`warnings.json`) backs a TUI pane that surfaces `allow_auto_merge` and other pre-flight issues across restarts. `warnings.Record/Clear` integrated into engine warning sites.
- **Engine-mechanized `Closes #N` linkage** (#841). Every Fabrik-created PR body now starts with `Closes #N\n\n` as a hard guarantee â€” prepended by the engine, not assembled by the Implement skill. Linkage drift detection follows.
- **Watch fixes** (#848). `worktreeDir` now correctly includes `<owner>-<repo>` segment for multi-repo boards; status bar split into two zones to fix truncation; closed-issue resume path documented.

## Fixes

- **Supplant feedback loop** (#861). `engine/poll.go` no longer cancels in-flight per-issue contexts when a poll cycle re-encounters an active worker. The buggy behavior burned ~263â€“320 supplant events per hour on the maintainer's instance for ~11 hours; no stage could complete a turn under v0.0.68.
- **Auto-merge convergence panic on cold-start** (#852). `checkAutoMergeConvergence` now calls `ensureRepoReady` before any `worktreesFor` call. Without the guard, a fresh start with a budget-exhausted yolo PR would panic in `pauseForConvergenceFailed`.
- **`isProbeOnlyTerminal` worktree-existence check** (#858). The probe-only predicate now requires the on-disk worktree to be absent before marking an item terminal. Closed items in a cleanup stage with worktree still present now proceed through deep-fetch + `processItem` so `cleanup_worktree` actually runs.
- **`pid <= 0` guards in kill paths** (#853 follow-up). `killProcGroup` and `killProcGroupGraceful` reject `pid <= 0` before any `syscall.Kill`. Prevents accidental SIGTERM to the daemon's own process group (`syscall.Kill(-0, ...)` semantics).
- **`issueCtxs.Delete` pointer-equality guard** (#853 follow-up). On supplant-cancel raced with a new dispatch, the exiting goroutine no longer deletes the new entry. Holder pointer equality check prevents stranded contexts.
- **`--kill-grace-sigint=0s` honored at engine level**. Setting the SIGINT grace to zero skips SIGINT entirely rather than firing it back-to-back with SIGTERM.
- **Warnings file lifecycle** (#843 follow-up). `lastMtime` resets when the file disappears so re-creation is detected; re-record preserves `first_seen` and `dismissed` while updating all other entry fields.
- **Owner/repo validation tightened** (#843 follow-up). Closed PRs skipped in heal path; review feedback addressed.

## Upgrading

```bash

# Or download directly (auto-detects OS and architecture)

  --pattern "fabrik_*_$(uname -s | tr A-Z a-z)_$(uname -m | sed 's/x86_64/amd64/' | sed 's/aarch64/arm64/').tar.gz" \
  -O - | tar xz
```
