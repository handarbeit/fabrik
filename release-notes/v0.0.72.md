# Fabrik v0.0.72

## Summary

A follow-up to the v0.0.71 merge-train release that closes a configuration gap and a version-reporting bug. The merge train and its batch-tuning knobs are now settable in `.fabrik/config.yaml` (`merge_train: on`) — previously they were the only major settings reachable *only* via CLI flag or env var, which surprised operators who keep persistent config in the file. And `fabrik --version` now reports the correct tag for `go install pkg@vX.Y.Z` installs instead of misleadingly printing `dev`. Rounded out with the documentation that trailed the v0.0.71 merge-train launch.

## Features

- `merge_train` (and the merge-train batch-tuning knobs — `max_batch_size`, `max_bisect_validations`, trial-window caps) are now settable in `.fabrik/config.yaml`, with the standard precedence `--merge-train` flag > `FABRIK_MERGE_TRAIN` env > `merge_train:` config key > default `off` (#992)

## Fixes

- `fabrik --version` reports the installed tag (e.g. `v0.0.72`) for `go install pkg@vX.Y.Z` builds instead of `dev` — it now falls back to `debug.BuildInfo.Main.Version` when no VCS revision is stamped (#994)

## Improvements

- Documentation catch-up for the v0.0.71 merge train / merge queue: README and docs-index feature coverage, the runaway-guard flag rows, and the `FABRIK_MERGE_QUEUE` env-var reference

## Upgrading

```bash
# Auto-upgrade from a running Fabrik instance
# Fabrik checks for new releases each poll cycle and upgrades automatically with --auto-upgrade

# Or download directly (auto-detects OS and architecture)
gh release download --repo handarbeit/fabrik \
  --pattern "fabrik_*_$(uname -s | tr A-Z a-z)_$(uname -m | sed 's/x86_64/amd64/' | sed 's/aarch64/arm64/').tar.gz" \
  -O - | tar xz
```
