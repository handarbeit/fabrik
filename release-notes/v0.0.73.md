# Fabrik v0.0.73

## Summary

Two engine robustness fixes that close "stranded forever" gaps when a single one-shot GitHub mutation fails. A spawned child issue whose initial board placement fails is now retried every poll instead of being lost, and the merge train's one-at-a-time landing now retries a failed member-issue close instead of leaving the issue open in the Done column. Both extend the durable-marker + settle-scan pattern introduced in v0.0.71/v0.0.72 for the no-work-needed path.

## Fixes

- **Spawned child board-placement is now retried** — if a child issue's initial project-board Status placement fails (API error, missing status-field metadata, or no suitable column), a durable `fabrik:awaiting-placement` marker is set and the placement retried every poll, rather than the child being stranded in Backlog forever (#984)
- **Merge-train singleton member-close is now retried** — `landSingleton`'s member-issue close, if it fails after the PR merge and Done-move, now carries a durable `fabrik:awaiting-member-close` marker and retries every poll instead of leaving the member issue open in the Done column (most likely on a non-default base branch, where GitHub's own `Closes #N` never auto-fires) (#985)

## Upgrading

```bash
# Auto-upgrade from a running Fabrik instance
# Fabrik checks for new releases each poll cycle and upgrades automatically with --auto-upgrade

# Or download directly (auto-detects OS and architecture)
gh release download --repo handarbeit/fabrik \
  --pattern "fabrik_*_$(uname -s | tr A-Z a-z)_$(uname -m | sed 's/x86_64/amd64/' | sed 's/aarch64/arm64/').tar.gz" \
  -O - | tar xz
```
