# Fabrik v0.0.70

## Summary

Four themes:

1. **PR-settling unified across three recovery paths.** A new `settlePRMergeState` primitive consolidates what were previously three independent code paths — auto-merge convergence, paused-item merged-PR recovery, and CI/review gate clearance — into a single `runValidatePRTerminalAdvance` owner. One source of truth for "given a PR's current state, what should Fabrik do next?" One inline `advanceToNextStage` call when a PR has merged. Closes the family of bugs where one path advanced labels correctly but its sibling didn't, leaving issues stuck at Validate or Review with merged PRs.

2. **Self-repair backstops landed.** A board-drift detection scan auto-advances items whose `stage:X:complete` label has gotten ahead of the board column (closes the 30-60 min lag operators saw in the 2026-06-16 stuck-Validate outbreak). The R4 paused-item recovery loop now covers `wait_for_reviews` symmetrically with `wait_for_ci`. Worker-liveness clears stranded `WorkerEntered` handles when the Claude process dies without context-cancel — no more 3-hour silent stalls.

3. **CI gate is tighter on the post-push window.** A new dwell guard blocks the gate-clearance race where `mergeable_state` returns `""` or `"unknown"` immediately after a Validate force-push, before GitHub has computed mergeability. Previously this transient window caused false "no CI configured" clearances and convergence-budget pauses.

4. **Operational hardening.** Log-retention janitor periodically prunes `.fabrik/logs/` by age and total size (`FABRIK_LOG_RETENTION_DAYS`, `FABRIK_LOG_MAX_BYTES`). Validate and Review skills now prompt Claude to run the project's dependency-install command after rebase — closing the class of stale-deps test failures (cartridge version mismatch, etc.). Comment-processing history no longer dedup-collapses repeat invocations.

## Features

- **`settlePRMergeState` primitive + `runValidatePRTerminalAdvance` single owner** (#898). New REST method `FetchPRMergeableFields` returns the combined mergeable + mergeable_state + headSHA + merged + state in one call. The primitive interprets these into a typed `PRSettleResult` (Merged, Mergeable, Behind, Conflicting, Pending, Blocked, etc.); `runValidatePRTerminalAdvance` is the single dispatcher that consults the result and decides between advance/wait/rebase/pause. Replaces three previously separate paths.
- **Convergence-paused recovery loop (R4b)** (#873). Items paused by `pauseForConvergenceFailed` (which removes `fabrik:auto-merge-enabled` but adds no `fabrik:awaiting-ci`) are now auto-recovered when their linked PR merges externally. Sits beside R4; same lock-aware shape.
- **R4 expanded to cover `wait_for_reviews`** (#874). The R4 paused-item recovery loop, previously only handling `wait_for_ci` gating, now symmetrically handles paused items whose `wait_for_reviews: true` stage was gated when the PR merged externally. Adds `stage:Review:complete` and advances correctly.
- **Worker-liveness staleness detection** (#872). New `WorkerStaleTimeout` config (default 5 min); shared `isWorkerStale` helper. The janitor + catch-up dispatch consult the same staleness predicate. A `WorkerEntered` whose PID is dead AND heartbeat is older than the timeout is cleared synthetically. No more multi-hour stalls from Claude crashes that bypass context-cancel.
- **Post-push dwell guard for the CI gate** (#871). New `PostPushDwell` engine config (default 90s). `checkCIGate` no longer falsely clears as "no CI configured" when `mergeable_state ∈ {"", "unknown"}` AND `time.Since(LastHeadSHAUpdate) < PostPushDwell`. Closes the race where Validate's rebase+push triggered immediate gate-clear before GitHub had computed mergeability.
- **Log-retention janitor**. New `LogRetentionDays` (default 14d) and `LogMaxBytes` (default 1 GiB) config; periodic `pruneLogs` background task. `.fabrik/logs/` no longer grows unbounded.
- **Validate + Review skills prompt project dependency install after rebase** (#875). Both skills now consult the project's `CLAUDE.md` for the install command (e.g., `pnpm install`, `cargo build`, etc.) and run it between rebase and test execution. Closes the class of test failures where `main` advanced past a dependency-bump and the worktree's installed deps no longer match the rebased lockfile.
- **Validate pre-completion gate against non-mergeable PRs** (#828). Validate skill is now instructed to emit `FABRIK_BLOCKED_ON_INPUT` (not `FABRIK_STAGE_COMPLETE`) when the linked PR has `mergeable_state ∈ {dirty, conflicting}` or a rebase was aborted mid-flight. Combined with the existing engine handling for the marker, the issue pauses cleanly instead of marking Validate complete on a non-mergeable PR.

## Fixes

- **`advance to Done inline when PR already merged in catch-up paths`** (#873). Items observed merged in the convergence catch-up now advance to Done within the same poll cycle, not delayed by 30-60 min until some other reconciliation fires. Includes a startup drift scan that emits warnings when `stage:X:complete` is present but the board column doesn't match.
- **`checkMergeabilityGate` clears (not blocks) on PRMergeBlocked**. Previously, a PR in the `blocked` state (e.g., awaiting required review) was treated as in-flight gate-blocked; the new settle primitive correctly classifies this as terminal-clear so the merge handoff fires cleanly.
- **Re-confirm merged state via single-PR endpoint before pausing** (#897). The convergence-paused recovery loop no longer trusts the boardcache's potentially-stale `pr.Merged` flag; it re-fetches via `e.client.FetchLinkedPR` before applying the pause.
- **Honor `FABRIK_STAGE_COMPLETE` on non-zero exit; record error separately**. Stage completion markers emitted before a non-zero exit are no longer discarded. The error is recorded as a separate history entry; the completion still applies if the marker was present.
- **History preserves all invocations without dedup collapse**. Stage invocation history was incorrectly collapsing consecutive identical entries; now each invocation is preserved for accurate audit.
- **`fabrik-validate` skill uses dynamic base branch and project-agnostic build commands**. Previously hardcoded `main` and `go build ./...`; now reads the PR's `baseRefName` and defers to project's CLAUDE.md for build/test commands.
- **`itemstate.LinkedPRState` tracks `LastHeadSHAUpdate`** (#871). New field set on every `PRHeadSHAUpdated` event; the post-push dwell guard consults this.
- **`pruneLogs` Gemini review findings addressed**. Three findings on the log janitor implementation cleared in follow-up commits.
- **Boardcache `ItemKey` uses resolved owner/repo in paused-item recovery**. Previously could miss items whose `item.Repo` field was non-canonical.

## Improvements

- **Phase 1 catch-up loop decomposed into ordered handler list**. The previously monolithic Phase 1 dispatch loop in `engine/poll.go` is now expressed as a sequence of named handler functions with explicit ordering: easier to test, easier to add new handlers without altering control flow.
- **Skill consistency fixes**: kill-grace docs (v0.0.69 leftover review), revalidate label list correction in README, dep-install step doc additions.
- **State-machine documentation reconciled against the code** (2026-06-17 audit). `docs/state-machine.md` invariants and code paths re-aligned; mismatches noted in ADR-056.
- **Convergence catch-up rebase-cycle bound**. The auto-merge convergence path now respects `MaxRebaseCycles` to prevent runaway rebase loops on stuck PRs.

## Upgrading

```bash
# Auto-upgrade from a running Fabrik instance
# Fabrik checks for new releases each poll cycle and upgrades automatically with --auto-upgrade

# Or download directly (auto-detects OS and architecture)
gh release download --repo handarbeit/fabrik \
  --pattern "fabrik_*_$(uname -s | tr A-Z a-z)_$(uname -m | sed 's/x86_64/amd64/' | sed 's/aarch64/arm64/').tar.gz" \
  -O - | tar xz
```
