# Fabrik v0.0.68

## Summary

Three themes:

1. **Post-Validate convergence is now atomic.** Yolo issues use GitHub's native auto-merge instead of Fabrik's poll-merge loop. When CI passes, required reviews are in, and the PR is mergeable, GitHub merges in a single transaction — no more `HTTP 405` race spam, no more "CI fix cycle limit reached" on a PR whose only real problem is that main moved during CI. A wall-clock convergence budget replaces the per-gate cycle limits as the post-Validate backstop, and when it exhausts the pause comment names the *actual* state (commits-behind-main, latest CI conclusion, mergeable status) rather than the misleading per-gate message. `fabrik:cruise` is explicitly excluded — cruise users still get the "test the branch yourself, merge by hand" workflow they want.

2. **Auto-merge fallback covers every terminal GitHub state.** GitHub's `enablePullRequestAutoMerge` mutation refuses to enable auto-merge whenever the PR is in a state where auto-merge has nothing useful to do — `CLEAN`, `UNSTABLE`, and likely others. The engine now treats any non-repo-level error from that call as "fall back to a direct merge attempt," which bounds the failure surface to genuine non-mergeability (where the existing rebase/CI-fix gates take over). A startup pre-flight check warns operators when `allow_auto_merge` is disabled on a managed repo, with the exact fix command, so the "PRs aren't merging and I don't know why" support case stops existing.

3. **Customisation-protection regression fixed.** A pre-v0.0.64 customised `.fabrik/plugin/` directory could be silently overwritten on first contact with a newer binary, because the migration treated the customised disk-hash as the baseline. Now corrected, with `tools/print-plugin-hash` for computing embedded hashes per release. Companion post-mortem of the v0.0.66 broken-`addBlockedBy` mutation is captured as a spec under `specs/827-postmortem-v0.0.66/`.

## Features

- **Convergence budget + GitHub native auto-merge for yolo issues** (#829). Replaces the legacy `MergePR` poll loop for the post-Validate yolo path. `enablePullRequestAutoMerge` is called once at Validate-complete; `fabrik:auto-merge-enabled` is applied as the idempotency guard and budget-start anchor; `checkAutoMergeConvergence` monitors the in-flight convergence and advances to Done when the PR merges. Budget tunable via `FABRIK_CONVERGENCE_BUDGET` (default 30m; `0` disables the bounded budget for legacy cycle-count behavior).
- **`allow_auto_merge` startup pre-flight check** (#832). On startup, the engine queries each managed repo's `allow_auto_merge` setting and prints a loud per-repo warning if disabled — naming the repo and the exact `gh api -X PATCH ... -f allow_auto_merge=true` fix command. `USER_GUIDE.md` gains an "Allow auto-merge" callout in the Yolo Mode section.
- **`tools/print-plugin-hash` utility** (#820 support). Computes the canonical hash of the embedded plugin tree for use by the release script's installed-version stamping. Wired into `scripts/cut-release.sh` so new releases bake in the correct hash automatically.

## Fixes

- **Generalize auto-merge fallback to cover ALL terminal `enablePullRequestAutoMerge` errors** (#835). #831 shipped a narrow `"clean status"` fallback; #835 generalises to any non-`ErrAutoMergeNotEnabled` error, so `UNSTABLE` (and any future GitHub variant) also falls back to a direct merge attempt rather than getting stuck open forever. Repo-level `Auto merge is not allowed` stays special-cased because retrying is futile without operator action.
- **`fabrik:auto-merge-enabled` removal on merge** (#829). Engine observes the PR merging, advances to Done, and removes the engine-internal label so the issue's terminal state is clean.
- **Pause on user-disabled auto-merge mid-flight** (#829). When the operator clicks "Disable auto-merge" in the GitHub UI, the engine observes `auto_merge_enabled=false`, treats it as opt-out, and pauses the convergence loop with a one-line takeover comment — no auto-revert label changes.
- **`cruise` wins over `yolo` for auto-merge at Validate** (#829). Per the existing precedence rule, an issue with both labels follows cruise behavior: no auto-merge enabled, no convergence budget, no label applied. Docs corrected to match.
- **`ErrAutoMergeAlreadyClean` handled in `dispatchRebaseReinvoke` fallback path**. Prevents a wrong-path retry when the rebase reinvoke encounters an already-merged PR.
- **Pre-v0.0.64 customisation protection** (#820). The migration no longer treats a customised `.fabrik/plugin/` disk-hash as the baseline; pre-existing operator customisations survive upgrade. Companion: `ComputeDiskVersion` returns `""` (not a phantom hash) for an empty plugin dir.
- **v0.0.66 broken-mutation post-mortem** (#827). `addBlockedByIssue` → `addBlockedBy` analysis filed as a spec under `specs/827-postmortem-v0.0.66/`; prevention measures called out.
- **Doc fix**: cruise-vs-yolo distinction for spawned child labels clarified in the plugin SKILL.

## Improvements

- **`logf` for auto-merge warnings**: routed through `e.logf` for TUI safety so multi-issue concurrent output stays readable.
- **State-machine doc**: section 5.4 documents the generic auto-merge fallback and the `fabrik:auto-merge-enabled` label lifecycle.
- **Marketing site**: Quickstart moved up to immediately follow the hero; Webhook Mode feature card added.

Two new e2e regression tests (`TestYoloAutoMergeLabel`, `TestConvergenceRace`) landed under `tests/e2e/` — they caught #831, #832, and #835 deterministically before any could hit a production user. Harness hardened to tolerate transient `gh` errors during long polls. Spec dir naming standardized to include issue number; #799 spec body ported verbatim. `print-plugin-hash` binary `.gitignore`d. ## Upgrading

```bash
# Auto-upgrade from a running Fabrik instance
# Fabrik checks for new releases each poll cycle and upgrades automatically with --auto-upgrade

# Or download directly (auto-detects OS and architecture)
gh release download --repo handarbeit/fabrik \
  --pattern "fabrik_*_$(uname -s | tr A-Z a-z)_$(uname -m | sed 's/x86_64/amd64/' | sed 's/aarch64/arm64/').tar.gz" \
  -O - | tar xz
```

**Operator action required on first upgrade:** if `allow_auto_merge` is disabled on any managed repo, yolo issues will reach Validate complete but their PRs will not merge. Fabrik's new startup check will print a per-repo warning naming the fix command. To enable proactively:

```bash
gh api -X PATCH repos/<owner>/<repo> -f allow_auto_merge=true
```
