# Fabrik v0.0.1 — Initial Release

The first release of Fabrik, an automated Claude Code SDLC driver powered by GitHub Issues and Projects.

## What's Included

### Pipeline Engine
- **7-stage pipeline**: Specify → Research → Plan → Implement → Review → Validate → Done
- Concurrent worker dispatch with configurable parallelism
- Cooldown retries with max-retry escalation
- `FABRIK_STAGE_COMPLETE` and `FABRIK_BLOCKED_ON_INPUT` markers
- `fabrik:yolo` label for per-issue auto-advance and auto-merge
- Session resume across retries

### Plugin Skills
- Stage workflow skills for each pipeline stage
- Comment processing skills with stage-specific behavior
- Complexity assessment in Research (recommends `model:opus` for complex issues)
- Installed via `fabrik init`, loaded via `--plugin-dir`

### TUI Dashboard
- Real-time In Progress and History panes
- Completion status icons: ✓ completed, ↻ retry, ? input needed, ✗ error
- Keyboard navigation with Tab/j/k, log viewer (l), history management (c/C)
- Persistent history in `.fabrik/history.json`
- Configurable terminal app (ghostty, iterm2, kitty, alacritty, warp)

### Multi-Repo Support
- User and org-level project board queries
- Per-repo worktree managers with bare clone support
- Job-control directory pattern (run Fabrik outside any git repo)
- Repo filtering for multi-repo project boards

### Context & Staleness
- Context files (`.fabrik-context/`) written before each invocation
- Stage comment rewriting (living documents via UpdateComment)
- Codebase changes tracking between stages
- `update_issue_body` gating (only Specify can modify the spec)

### Observability
- Raw JSON output logging to `~/.fabrik/logs/`
- `fabrik _stream-filter` for human-readable log viewing
- `fabrik _debug-server` for HTTP-based debugging
- Rate limit monitoring (two-phase GraphQL optimization)
- `fabrik watch` subcommand for per-issue monitoring

### Configuration
- `.fabrik/config.yaml` for project-level settings
- `.env` for secrets only
- `fabrik init` with interactive setup
- `the download command below` to refresh plugin skills
- Auto-upgrade from origin/main when idle

## Installation

```bash
# Download and extract
tar xzf fabrik_0.0.1_darwin_arm64.tar.gz

# Or build from source
go build -o fabrik .
```

## Getting Started

```bash
fabrik init          # scaffold stages, plugin, and config
fabrik               # start processing issues
fabrik --tui         # with the terminal dashboard
```

See the [User Guide](https://handarbeit.github.io/fabrik/USER_GUIDE) for full documentation.


> **Note:** Binary self-update via `fabrik upgrade` works from v0.0.8 onwards. For this version, download manually:
> ```bash
> gh release download --repo handarbeit/fabrik --pattern 'fabrik_*_darwin_arm64.tar.gz' -O - | tar xz
> ```
